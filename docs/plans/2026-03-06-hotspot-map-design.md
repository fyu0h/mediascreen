# 热点区域地图模块设计方案

**日期：** 2026-03-06
**状态：** 已批准
**目标：** 在全球新闻源地图同位置增加热点区域地图，支持自动切换和手动配置

---

## 1. 整体架构

### 前端结构
```
地图容器
├── 新闻源地图（现有）
├── 热点区域地图（新增）
├── 切换控制器
│   ├── 滑块开关
│   └── 地图标题（动态显示）
└── 热点详情卡片（叠加层）
```

### 后端结构
```
MongoDB 新增集合：hotspot_zones
├── 字段：name, description, risk_level, coordinates, videos, created_at, updated_at

新增路由：
├── /hotspot-admin（管理页面）
├── /api/hotspots/*（CRUD API）
└── /api/hotspots/videos/upload（视频上传）
```

### 实现方案
采用**双地图实例切换方案**：
- 创建两个独立的 Leaflet 地图实例
- 通过 CSS 控制显示/隐藏
- 使用 opacity 和 visibility 实现淡入淡出动画
- 地图瓦片源：PMTiles（OpenFreeMap 回退）

**优势：**
- 两个地图独立渲染，互不干扰
- 切换流畅，动画效果好
- 易于维护和扩展

## 2. 地图切换机制

### 自动轮播逻辑
```
初始状态：显示新闻源地图
├── 5秒后自动切换到热点地图（淡出淡入）
├── 再5秒后切换回新闻源地图
└── 循环往复

用户交互：
├── 点击滑块 → 立即切换 + 暂停自动轮播
├── 30秒无操作 → 恢复自动轮播
└── 操作计时器重置条件：点击滑块、点击热点标记
```

### CSS 动画
- 使用 `opacity` 和 `transition` 实现淡入淡出
- 过渡时间：0.8秒
- 非活动地图设置 `pointer-events: none` 防止误触

### 滑块位置
- 放在地图卡片标题栏右侧
- 显示当前地图名称："全球新闻源分布" ↔ "全球热点区域"

## 3. 热点区域数据结构

### MongoDB 集合：`hotspot_zones`

```javascript
{
  _id: ObjectId,
  name: String,              // 热点名称，如"加沙地带冲突"
  description: String,       // 态势描述，支持多行文本
  risk_level: String,        // 风险等级："high" | "medium" | "low"
  coordinates: [             // 多边形坐标数组
    [lat, lng],
    [lat, lng],
    ...
  ],
  videos: [                  // 视频文件数组
    {
      filename: String,      // 文件名
      path: String,          // 服务器路径
      title: String,         // 视频标题（可选）
      upload_time: Date
    }
  ],
  created_at: Date,
  updated_at: Date,
  enabled: Boolean           // 是否启用显示
}
```

### 多边形样式配置
- **高风险：** 红色填充 `rgba(255, 71, 87, 0.3)` + 红色边框 `#ff4757`，边框宽度 2px
- **中风险：** 橙色填充 `rgba(255, 165, 2, 0.3)` + 橙色边框 `#ffa502`，边框宽度 2px
- **低风险：** 黄色填充 `rgba(255, 234, 0, 0.3)` + 黄色边框 `#ffea00`，边框宽度 2px
- **鼠标悬停：** 透明度增加到 0.5，边框加粗到 3px

## 4. 热点详情卡片设计

### 卡片布局
```
┌─────────────────────────────────┐
│ [×] 热点名称              风险标签 │
├─────────────────────────────────┤
│ 态势描述文字（多行）              │
│                                 │
├─────────────────────────────────┤
│ 更新时间：2026-03-06 14:30      │
├─────────────────────────────────┤
│ ┌─────────────────────────┐     │
│ │                         │     │
│ │   视频播放区域           │     │
│ │   (16:9 比例)           │     │
│ │                         │     │
│ └─────────────────────────┘     │
│ ◄  ● ● ● ●  ►  (1/4)           │
└─────────────────────────────────┘
```

### 卡片特性
- **位置：** 地图右上角，距离边缘 20px
- **宽度：** 400px
- **最大高度：** 600px（超出滚动）
- **背景：** 半透明深色 `rgba(10, 20, 40, 0.95)` + 模糊效果
- **边框：** 科技蓝色发光边框
- **关闭按钮：** 右上角 × 图标

### 视频轮播
- 横向滑动切换（触摸/鼠标拖拽）
- 左右箭头按钮
- 底部指示器圆点（显示当前位置）
- 视频格式支持：mp4, webm, mov
- 自动播放：否（用户点击播放）

### 风险标签样式
- **高风险：** 红色徽章 "高风险"
- **中风险：** 橙色徽章 "中风险"
- **低风险：** 黄色徽章 "关注"

## 6. API 端点设计

### 热点 CRUD API

| 方法 | 端点 | 功能 | 请求体 |
|------|------|------|--------|
| GET | `/api/hotspots` | 获取热点列表 | query: `?enabled=true&risk_level=high` |
| GET | `/api/hotspots/<id>` | 获取单个热点详情 | - |
| POST | `/api/hotspots` | 创建新热点 | `{name, description, risk_level, coordinates, enabled}` |
| PUT | `/api/hotspots/<id>` | 更新热点 | `{name, description, risk_level, coordinates, enabled}` |
| DELETE | `/api/hotspots/<id>` | 删除热点（含视频文件） | - |

### 视频管理 API

| 方法 | 端点 | 功能 | 说明 |
|------|------|------|------|
| POST | `/api/hotspots/<id>/videos` | 上传视频 | multipart/form-data，返回视频信息 |
| DELETE | `/api/hotspots/<id>/videos/<filename>` | 删除视频 | 同时删除服务器文件 |
| GET | `/api/hotspots/videos/<filename>` | 获取视频文件 | 流式传输视频 |

### 视频存储
- **目录：** `static/uploads/hotspot_videos/`
- **文件名：** `{hotspot_id}_{timestamp}_{original_name}`
- **大小限制：** 100MB/文件
- **格式限制：** `.mp4`, `.webm`, `.mov`

### 响应格式
```json
{
  "success": true,
  "data": {
    "_id": "...",
    "name": "加沙地带冲突",
    "description": "...",
    "risk_level": "high",
    "coordinates": [[31.5, 34.4], ...],
    "videos": [
      {
        "filename": "abc123_video.mp4",
        "path": "/static/uploads/hotspot_videos/abc123_video.mp4",
        "title": "现场视频",
        "upload_time": "2026-03-06T14:30:00Z"
      }
    ],
    "enabled": true,
    "created_at": "...",
    "updated_at": "..."
  }
}
```

## 7. 技术实现要点

### 前端实现

#### 1. 地图切换动画
- 使用 CSS `transition: opacity 0.8s ease-in-out`
- JavaScript 控制 `display` 和 `opacity` 属性
- 切换前预加载热点数据，避免白屏

#### 2. 视频轮播
- 使用原生 JS 实现（不引入额外库）
- 触摸事件：`touchstart`, `touchmove`, `touchend`
- 鼠标事件：`mousedown`, `mousemove`, `mouseup`
- 滑动阈值：50px

#### 3. 自动轮播计时器
- 使用 `setInterval` + `setTimeout` 组合
- 用户交互时 `clearInterval`，30秒后重新启动
- 页面失焦时暂停，聚焦时恢复

### 后端实现

#### 1. 视频上传处理
- 使用 Flask `request.files` 接收文件
- 文件类型验证：检查 MIME type 和扩展名
- 文件大小验证：限制 100MB
- 异步保存：避免阻塞请求

#### 2. 安全措施
- **文件名消毒：** 移除特殊字符，防止路径遍历
- **SSRF 防护：** 视频 URL 不接受外部链接（仅上传）
- **权限控制：** 管理页面和 API 需要 admin 角色
- **文件清理：** 删除热点时同步删除关联视频文件

#### 3. 性能优化
- **热点数据缓存：** 前端缓存 5 分钟
- **视频流式传输：** 使用 `send_file` 支持 Range 请求
- **MongoDB 索引：** `enabled` 和 `risk_level` 字段
- **图片懒加载：** 视频封面延迟加载

### 错误处理
- **视频上传失败：** 显示错误提示，支持重试
- **地图加载失败：** 显示占位符和重新加载按钮
- **热点数据为空：** 显示"暂无热点区域"提示
- **网络超时：** 3秒超时，自动重试 2 次
