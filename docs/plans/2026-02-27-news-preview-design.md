# 新闻预览页面设计方案

> 日期：2026-02-27
> 状态：已批准

## 概述

用户点击新闻时，不再直接跳转原始链接，而是在新闻条目下方内嵌展开预览区域，后台实时爬取原始页面并通过 iframe srcdoc 渲染，同时提供「访问原始链接」按钮。

## 需求决策

| 维度 | 选择 |
|------|------|
| 展示形式 | 内嵌展开（手风琴效果） |
| 抓取时机 | 点击时实时抓取 |
| 展示内容 | 原始 HTML 渲染（iframe srcdoc） |
| 跳转按钮位置 | 顶部工具栏 |
| 实现方案 | 方案 A：iframe srcdoc + HTML 处理 |

## 后端 API

### 新增端点：`GET /api/news/preview`

**参数**：`url` — 要预览的新闻原始链接

**处理流程**：

1. 接收 `url` 参数，校验合法性（必须是 http/https）
2. 复用 `PluginCrawler.fetch_page()` 抓取原始页面 HTML
3. HTML 处理：
   - 注入 `<base href="原始域名">` 标签，确保相对路径资源正确加载
   - 移除 `<script>` 标签（安全考虑，防止恶意脚本执行）
   - 保留 `<style>` 和 `<link rel="stylesheet">`（保持样式）
   - 保留 `<img>` 标签（图片展示）
4. 返回 JSON：`{"success": true, "data": {"html": "处理后的HTML", "url": "原始URL"}}`

**错误处理**：

- URL 缺失或非法 → 400
- 抓取失败/超时 → 返回失败提示，前端展示 fallback 消息 + 原始链接按钮

## 前端交互与 UI

### 点击行为改造

- **当前行为**：`window.open(url, '_blank')` 直接跳转
- **改造后**：点击新闻条目 → 在该条目下方内嵌展开预览区域，再次点击收起

### 预览区域结构

```
┌─────────────────────────────────────────┐
│ 新闻条目（标题、来源、时间等）           │  ← 点击展开/收起
├─────────────────────────────────────────┤
│ ┌─ 工具栏 ─────────────────────────────┐│
│ │ 🔗 访问原始链接    ✕ 收起             ││
│ └───────────────────────────────────────┘│
│ ┌─ iframe (srcdoc) ────────────────────┐│
│ │                                       ││
│ │   原始页面 HTML 渲染内容              ││
│ │   (高度 500px，内部滚动)              ││
│ │                                       ││
│ └───────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

### 交互细节

- **加载状态**：展开后先显示加载动画，API 返回后替换为 iframe 内容
- **同时只展开一个**：点击新条目时自动收起之前展开的预览
- **收起方式**：再次点击标题，或点击工具栏「收起」按钮
- **工具栏**：固定在预览区域顶部，含「访问原始链接」和「收起」按钮
- **iframe 高度**：固定 500px，内部滚动
- **样式**：与现有科技深色主题保持一致

## 错误处理与边界情况

### 抓取失败 Fallback

工具栏始终可用，iframe 区域显示提示文字：「无法加载预览内容，请点击上方访问原始链接查看原文」

### 边界情况

| 场景 | 处理方式 |
|------|---------|
| URL 为空或无效 | 前端直接显示 fallback，不发请求 |
| 抓取超时（默认 15s） | 后端返回错误，前端显示 fallback |
| HTML 内容过大（>5MB） | 后端截断并提示，建议访问原始链接 |
| 重复点击同一条目 | 收起/展开切换，不重复请求 |
| 快速连续点击不同条目 | 收起前一个，展开新的，取消未完成请求 |
