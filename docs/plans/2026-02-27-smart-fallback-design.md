# 新闻预览智能回退链设计方案

> 日期：2026-02-27
> 状态：已批准

## 概述

当前新闻预览功能在部分网站（Cloudflare 保护、重度 SPA、付费墙）上无法成功提取正文内容。本方案引入智能回退链机制，确保任何网站都能为用户提供可用的预览结果。

## 回退链路

```
Level 1: requests.get() → 正文提取
  ↓ 失败/Cloudflare 检测
Level 2: crawl4ai 无头浏览器 → 正文提取
  ↓ 失败/内容过少（blocks < 3 或文本 < 100 字符）
Level 3: Playwright 全页截图 → Base64 内联返回
  ↓ 截图也失败
Level 4: 数据库缓存摘要（标题/来源/时间 + 原始链接按钮）
```

## 需求决策

| 维度 | 选择 |
|------|------|
| 回退策略 | 智能回退链（4 级） |
| 截图技术 | Playwright（复用现有 crawl4ai 无头浏览器） |
| 截图存储 | Base64 内联返回（无需文件管理） |
| 内容不足判断 | content blocks < 3 或纯文本 < 100 字符 |

## API 响应格式

### 正文模式（现有，Level 1/2 成功时）

```json
{
  "success": true,
  "data": {
    "type": "content",
    "title": "页面标题",
    "url": "原始URL",
    "content": [
      {"type": "paragraph", "text": "..."},
      {"type": "image", "src": "/api/proxy/image?url=...", "alt": "..."}
    ]
  }
}
```

### 截图模式（新增，Level 3 成功时）

```json
{
  "success": true,
  "data": {
    "type": "screenshot",
    "image": "data:image/png;base64,iVBOR...",
    "url": "原始URL"
  }
}
```

### 缓存模式（新增，Level 4 兜底）

```json
{
  "success": true,
  "data": {
    "type": "cached",
    "title": "文章标题",
    "source": "新闻源名称",
    "pub_date": "2026-02-27",
    "url": "原始URL"
  }
}
```

## 后端改造

### `/api/news/preview` 端点改造

1. **Level 1**: 保持现有 `requests.get()` + BeautifulSoup 正文提取逻辑
2. **Cloudflare 检测**: 保持现有标记检测逻辑
3. **Level 2**: 保持现有 crawl4ai 无头浏览器回退 + 正文提取
4. **内容质量检查**（新增）: 提取后检查 content blocks 数量和文本长度
5. **Level 3**（新增）: 使用 Playwright 截取全页截图，转 Base64 返回
6. **Level 4**（新增）: 从 MongoDB `news_articles` 集合中查询该 URL 的缓存信息

### 截图参数

- 视口宽度：1280px
- 视口高度：720px
- 格式：PNG
- 全页截图：是
- 超时：20 秒

## 前端改造

### `openNewsPreview()` 函数

根据 `data.type` 渲染不同内容：

- `"content"`: 保持现有正文渲染逻辑
- `"screenshot"`: 显示截图图片（可缩放、可滚动）
- `"cached"`: 显示简化的缓存信息卡片 + 原始链接按钮
